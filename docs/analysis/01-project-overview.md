## 二、项目背景与痛点分析

### 2.1 行业背景

#### 英语听力与拼写训练的市场需求

在全球化背景下，英语作为国际通用语言的重要性不言而喻。对于非英语母语的学习者来说：

- **听力理解**是语言学习的四大基本技能之一（听、说、读、写）
- **拼写能力**是书面表达的基础
- **听写结合**（Dictation）被广泛认为是提高听力和拼写的有效方法

#### 传统学习方式的局限性

根据 [requirement.md](../../requirement.md) 的需求分析和市场调研，现有的英语学习方式存在以下痛点：

| 痛点类别 | 具体问题 | 影响 |
|---------|---------|------|
| **互动性不足** | 传统听写练习缺乏即时反馈 | 学习者无法及时纠正错误，错误习惯固化 |
| **材料分散** | 学习资源散布在不同平台和媒介 | 难以系统化管理，学习效率低下 |
| **进度不透明** | 缺乏有效的学习进度追踪机制 | 学习者难以评估自己的提升情况 |
| **移动体验差** | 许多工具未针对移动设备优化 | 无法利用碎片化时间学习 |
| **个性化缺失** | "一刀切"的内容和难度 | 无法满足不同水平学习者的需求 |

### 2.2 技术背景

#### Web 技术的成熟度

2026年的 Web 技术栈已经能够支持构建高质量的交互式学习应用：

✅ **现代前端框架**：React 19、Vue 3、Svelte 等提供出色的开发体验  
✅ **语音合成能力**：Web Speech API 已被主流浏览器广泛支持  
✅ **Serverless 架构**：Netlify Functions、Vercel Functions 等降低后端运维成本  
✅ **BaaS 服务**：Supabase、Firebase 等提供开箱即用的数据库和认证服务  
✅ **PWA 技术**：支持离线访问和类原生应用体验  

#### 开源教育内容的可用性

大量优质的教育内容可以合法获取和使用：

- 新概念英语教材（经典教材，广泛使用）
- 公开的 Notion 文档（支持动态更新）
- 社区贡献的学习资源（如 Anki 卡片集）

## 三、项目定位与产品定义

### 3.1 产品定位

**一句话描述**：
> 一个可在手机和电脑上使用的现代化句子听写练习平台，通过智能化的输入验证、多数据源支持和个性化进度追踪，帮助用户高效提升英语听力与拼写能力。

#### 核心价值主张

| 维度 | 价值主张 | 对比传统方式 |
|------|---------|------------|
| **便捷性** | 打开浏览器即可使用，无需安装 | ✅ vs 桌面软件安装繁琐 |
| **智能化** | 即时反馈、自动评分、智能提示 | ✅ vs 人工批改耗时 |
| **灵活性** | 多数据源可选，支持自定义内容 | ✅ vs 固定教材限制 |
| **可追踪** | 详细的学习统计和进度记录 | ✅ vs 黑盒式学习 |
| **跨平台** | 手机、平板、桌面全适配 | ✅ vs 仅支持特定设备 |
| **低成本** | 免费使用（基于 Netlify 免费额度） | ✅ vs 商业软件付费 |

### 3.2 目标用户画像

#### 主要用户群体

**👨‍🎓 学生群体（核心用户）**

- **年龄**：12-25岁
- **场景**：课堂辅助学习、课后自主练习、考试备考
- **痛点**：需要大量的听写练习但缺乏有效工具
- **期望**：操作简单、反馈及时、能跟踪进步
- **使用频率**：高频（每天或隔天）

**👩‍💼 职场人士（重要用户）**

- **年龄**：25-40岁
- **场景**：商务英语提升、出国准备、职业发展
- **痛点**：时间碎片化，需要利用通勤等零碎时间
- **期望**：移动端友好、快速启动、离线可用
- **使用频率**：中频（每周2-3次）

**👨‍🏫 教育工作者（内容创作者）**

- **角色**：教师、培训机构讲师
- **场景**：为学生定制练习内容、管理班级学习进度
- **痛点**：难以批量创建和管理练习材料
- **期望**：方便的内容录入工具、学生数据统计
- **使用频率**：低频但深度使用

#### 用户使用场景示例

根据 [requirement.md](../../requirement.md#5-使用场景示例) 中描述的两个典型场景：

**场景一：教师布置作业**

```
1️⃣ 教师准备阶段
   - 将一组英语句子写入 sentences.json 文件
   - 或维护一个 Notion 页面存放练习句子
   - 通过 Netlify 自动部署网站

2️⃣ 学生练习阶段
   - 通过手机或电脑访问网站
   - 选择对应的数据源（本地文件或Notion）
   - 进行听写练习，获得即时反馈

3️⃣ 效果
   - 无需安装任何软件
   - 内容更新只需修改 JSON 或 Notion
   - 学生随时随地进行练习
```

**场景二：内容创作者运营**

```
1️⃣ 创作者维护阶段
   - 在 Notion 上持续更新练习句子库
   - 可以按主题、难度分类组织内容
   - 支持多媒体扩展（音频、图片）

2️⃣ 用户学习阶段
   - 访问固定URL即可获取最新内容
   - 无需重新部署或下载更新
   - 自动同步最新的练习材料

3️⃣ 效果
   - 内容实时更新，保持新鲜感
   - 降低维护成本
   - 提升用户体验
```

### 4.1 项目目标层级

根据需求的优先级和实现复杂度，将目标分为三个层次：

#### 🎯 层级一：核心目标（MVP - Minimum Viable Product）

**必须达成**的基本功能，构成产品的最小可用版本：

| # | 目标 | 详细说明 | 验收标准 |
|---|------|---------|---------|
| G1 | **多数据源支持** | 支持从本地JSON文件和Notion公开文档读取句子 | 至少2种数据源可用且切换流畅 |
| G2 | **语音播放功能** | 提供TTS朗读当前句子，支持语速调节 | 主流浏览器（Chrome/Safari/Firefox）均可正常播放 |
| G3 | **逐词输入与判断** | 用户按词输入，系统自动比对并给出反馈 | 忽略大小写/空格/标点差异，准确率100% |

| G5 | **免费自动化部署** | 通过GitHub连接Netlify实现自动构建发布 | Push到main分支后10分钟内自动部署完成 |

## 五、约束条件与非功能需求

### 5.1 技术约束

根据 [requirement.md](../../requirement.md#4-非功能需求) 明确的非功能需求：

| 约束类型 | 具体要求 | 理由 |
|---------|---------|------|
| **部署方式** | 必须通过 GitHub + Netlify 实现自动化部署 | 降低运维成本，简化发布流程 |
| **后端架构** | 除Netlify Functions外不得依赖独立服务器 | 符合Serverless架构理念，控制成本 |
| **离线能力** | 使用本地数据源时需支持离线运行 | 提升用户体验，应对网络不稳定场景 |
| **安全性** | 不得在前端暴露私有凭证（密钥、令牌） | 防止敏感信息泄露 |
| **加载速度** | 句子内容应在页面加载后尽快可用 | 减少用户等待焦虑 |

### 5.2 业务约束

| 约束类型 | 具体要求 | 理由 |
|---------|---------|------|
| **成本控制** | 利用Netlify免费额度运行（每月100GB带宽、125万次函数调用、300分钟构建时间） | 项目初期零成本运营 |
| **内容合规** | 使用新概念英语等内容时需注意版权问题 | 法律风险规避 |
| **数据隐私** | 用户学习数据需符合GDPR等法规要求 | 合规性保障 |

### 5.3 团队资源约束

假设团队规模为 **3-5人**（参考 [06-项目规划与团队](./06-project-planning.md)）：

| 资源类型 | 约束 | 应对策略 |
|---------|------|---------|
| **人力** | 小型团队，需要一人多职 | 采用成熟技术栈降低学习成本；完善文档减少沟通成本 |
| **时间** | 需要在合理时间内交付MVP | 优先实现核心功能；采用敏捷迭代 |

#### 技术性排除
- ❌ **原生移动应用**（PWA已足够，React Native留待V2.0）
- ❌ **独立服务器/自建数据库**（违背Serverless原则）
- ❌ **实时通信/WebSocket**（当前无实时协作需求）
- ❌ **区块链/NFT/代币**（与教育场景无关）
- ❌ **VR/AR沉浸式体验**（技术过早熟，成本过高）

#### 内容性排除
- ❌ **自制版权内容**（避免法律风险，使用开源/授权内容）
- ❌ **多语言支持**（除中英文UI外，暂不支持其他语言教学内容）
- ❌ **专业考试题库**（如雅思/托福真题，涉及版权问题）

### 7.3 边界模糊区（Maybe Later）🤔

这些功能**可能在未来版本**实现，但不在当前计划中：

- 🤔 AI驱动的错因分析和个性化推荐
- 🤔 学习数据分析仪表盘（高级可视化）
- 🤔 第三方API开放平台
- 🤔 插件/扩展系统
- 🤔 与Anki卡片格式兼容
- 🤔 国际化（i18n）多语言UI
- 🤔 暗色模式（Dark Mode）
