# Testing Strategy Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Establish a comprehensive, maintainable test suite covering services, hooks, and components with ≥70% code coverage.

**Architecture:** Three-layer testing strategy:
1. **Unit Tests** - Services and pure utility functions with full mocking
2. **Hook Tests** - React custom hooks with render hooks from Testing Library
3. **Component Tests** - React components with user interactions simulated via user-event
4. **E2E Tests** (optional future) - Playwright for critical user flows

**Tech Stack:** Vitest, @testing-library/react, @testing-library/user-event, @playwright/test

### Key Issue: flashcardService.test.js

**Root Cause:** The `apiRequest()` and `historyApiRequest()` functions use **relative URLs** like `/api/flashcards/` which Node.js fetch cannot parse as valid URLs. Tests that call cloud API functions directly fail with:
```
TypeError: Failed to parse URL from /api/flashcards/
```

**Solution:** These tests need to be either:
1. Skipped (temporary) - until someone rewrites with proper async fetch mocking
2. Rewritten with vi.mock for the cloud functions - preventing real fetch calls
